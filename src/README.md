## 1. 어떤 병목을 다루는가

- **병목 Task**:  
과제 수행 과정 중 "제출 전 교수님이 제시한 요구사항(형식·내용·파일 구조)을 수작업으로 검증하는 단계"

- **Task 흐름**:
  1. 요구사항 확인  
  2. 자료 검색  
  3. 과제 작성  
  4. 제출 전 요구사항 검증 (**병목 지점**)  
  5. 제출  

- **빈도**:  
주 1~2회 / 1회당 약 10분

- **왜 병목인가**:
  - 요구사항이 자연어로 제공되어 항목별로 다시 정리하고 비교하는 과정이 반복됨  
  - 수작업 검증 과정에서 누락 및 실수가 발생할 가능성이 높음 (감점 리스크 존재)  
  - 단순 확인 작업임에도 조건이 많아 시간 대비 비효율적임  

## 2. 왜 AI Agent로 만들었는가

- **룰베이스/매크로/기존 도구로 안 되는 이유**:  
요구사항이 자연어로 표현되며, 사람마다 작성 방식과 표현이 달라 정형화된 규칙 기반 매칭이 어려움.  
또한 동일한 의미라도 다양한 표현이 존재하여 단순 키워드 기반 검증으로는 정확한 판단이 불가능함.

- **AI 판단이 필요한 지점**:  
요구사항을 구조화된 체크 항목으로 변환하는 과정 (자연어 이해)  
작성된 과제가 요구사항을 충족하는지 의미 기반으로 비교·판단하는 과정  
누락된 요소나 부족한 부분을 추론하여 피드백을 생성하는 과정  


## 3. Agent 구조

- **입력 → 처리 → 출력** (3줄 이내 다이어그램 또는 설명)
  -  요구사항 + 작성한 과제물  → 요구사항 파싱 + 과제물과 의미 기반 매칭  → 미반영 항목 추출 + 수정 필요 부분 리포트 생성
- **사용 도구**: (모델, 외부 API, 파일 시스템, MCP 서버 등)

- **핵심 제약 (Steering 요약)**:
  - 비용 상한: 
  - 도구 화이트리스트:
  - 사람 개입이 필요한 조건:
    -  요구사항이 반영되지 않은 부분을 수정하기 전 최종 검토
    -  판단이 모호한 항목 (ex. “충분히 설명되었는가”) 존재 시
